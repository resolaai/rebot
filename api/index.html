Rebot Developer API

アプリケーション詳細画面にて、「APIアクセス」を許可することで、API経由でのアクセスが可能となります。

共通仕様

■ エンドポイント 

https://rebot.jp/api/{app_unique_id}/{command}
  • {app_unique_id}は、「アプリケーション識別ID」です。アプリケーション詳細画面にて確認できます。
  • {command}は、各APIの機能を表します。
  • アクセスはhttpsのみ可能です。

■ リクエストヘッダ

APIコールする際は、リクエストヘッダへAPIアクセスキーを設定する必要があります。
APIアクセスキーは、アプリケーション詳細画面にて「APIアクセス」を許可することで取得できます。
APIアクセスキーが正しくセットされていない場合は、[400 Bad Request]を返します。

リクエストヘッダ 説明
x-access-key APIアクセスキー。

■ レスポンス

JSON形式で返却します。

アクション登録API

まだ開発中です

以下のJSON形式のリストにてアクションを受け取り、アクションの登録を行います。 まだ存在しないアクション名が指定された場合は新規追加を行い、既に存在するアクション名の場合は、当内容にて上書き更新されます。

■ エンドポイント 

/actions

リクエスト

■ リクエストメソッド 

POST

■ リクエストボディ

以下の値を持つJSONオブジェクトのリスト

パラメータ必須説明
update_type更新タイプ
1:新規追加（既にアクション名が存在する場合はスキップ）
2:部分追加（既にアクションが存在する場合は、text及びresponse_text, repliesのみを追加する）
3:置き換え（既にアクション名が存在する場合は置き換え、存在しない場合は追加）
*何も指定しない場合は"3:置き換え"となる
actionアクション名
指定されない場合は自動でアクション名が採番される
textsユーザが発言する内容のリスト
["発言1","発言2"]
responses返答内容のリスト
以下のJSONオブジェクトを持つリスト
response_text○返答内容
replies返答内容にボタンを含める場合のボタン
複数ある場合はカンマ区切りで指定

■ リクエストボディの例 

[
  {"update_type":"1", "action":"action1", "texts":["こんにちは"], "responses":[{"response_text":"こんにちは〜、お元気ですか？", "replies":["はい","いいえ"]}, {"response_text": "こんにちは"}]},
  {"update_type":"2", "action":"action2", "texts":["こんばんは","お疲れ様"], "responses": [{"response_text":"こんばんは〜"}]},
  {"update_type":"3", "action":"action3", "texts":["おはよう"], "responses":[{"response_text": "おはようございます"}]},
  {"texts":["自動アクション採番"]},
  {"texts":["自動アクション採番２"], "responses":[{"response_text":"自動でアクションを採番したよ"}]}
]

レスポンス

項目説明
result"OK"
updated更新されたアクションのリスト
["action2", "action3"]
skippedスキップされたアクションのリスト
["action1"]

ログ取得API

検索条件を指定して、ログデータの一覧を取得します。

■ エンドポイント 

/logs

リクエスト

■ リクエストメソッド 

GET

■ リクエストパラメータ

パラメータ 必須 説明
date_from 開始日付（yyyy-mm-dd)
値が無い場合は今日日付で検索
date_to 終了日付（yyyy-mm-dd)
値が無い場合は今日日付で検索
action アクション名を前方一致で検索
state ステート名を前方一致で検索

■ 例 

https://rebot.jp/api/xxxxxxxxxxxxxx/logs?date_from=2017-01-01&date_to=2017-03-31&action=sys

レスポンス

以下の情報を含むログ情報をリストで返します。

項目説明
userユーザを一意に識別するID
textユーザの会話
datetime日時
action認識したアクション名
stateステート名
response応答内容
result_jsonシステムで解析した情報
詳細はログ詳細画面を確認。
