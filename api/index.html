Rebot Developer API

ログインAPIを通して取得したAPIアクセスキーを使うことで、API経由で各種操作を行うことができるようになります。

API利用フロー

  1. ログインAPIでユーザ認証を行う
  2. サーバ側でAPIアクセスキーを発行し、返却する
  3. クライアント側でAPIアクセスキーを保存し、共通仕様の通りAPIアクセスキーを使い各種APIを利用する

共通仕様

■ エンドポイント 

https://rebot.jp/api/{command}
  • {command}は、各APIの機能を表します。
  • アクセスはhttpsのみ可能です。

■ リクエストヘッダ

APIコールする際は、リクエストヘッダへAPIアクセスキーを設定する必要があります。
APIアクセスキーは、ログインAPIにて認証を行うことで取得できます。
APIアクセスキーが正しくセットされていない場合は、[400 Bad Request]を返します。

リクエストヘッダ 説明
x-access-key APIアクセスキー
content-type application/json

■ レスポンス

JSON形式で返却します。　

項目説明
resultAPI処理結果
0 :成功
1 :不正なデータ（入力チェックエラー）
98: APIアクセスキーエラー
99:不正なリクエスト
data成功時に処理結果データを格納
messageオプション
エラー時のメッセージなど
error_fieldsオプション
入力チェックエラー時の項目とエラーメッセージのリスト
例）
[{'field':'email', 'message':'正しいメール形式ではありません'}]

※ 以下のAPI仕様書には、特に指定が無い場合は、data項目以下のみの定義を記述しています。

ログインAPI

■ エンドポイント 

/api/login
  • アクセスはhttpsのみ可能です。

リクエスト

■ リクエストメソッド 

POST

■ リクエストパラメータ

パラメータ必須説明
id○管理画面へのログインID
password○ログインパスワード

レスポンス

以下の項目を持つオブジェクト

項目説明
success1:ログイン成功
0:ログイン失敗
access_tokenログイン成功時に設定される
APIアクセスキー
ログインAPI以外の各種APIを利用する際に利用する
