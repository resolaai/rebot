アクション

ユーザが発言するであろう内容と、それに対する回答の設定を行います。
チャットボットに対してユーザから発言があった場合に、ここで定義したアクションの会話文との類似度を計算して、どのアクションに当たるのか判断をします。

基本設定項目

アクション基本情報タブ

  • アクション名:
    • アクションを一意に識別するためのID
    • 英数字及びアンダーバーのみ利用可能で、最初の文字は英数字で始まる必要があります。
  • ユーザが話かける会話文:
    • ユーザが発言するであろう内容を自然会話文そのまま入力します。
    • ユーザからチャットボットに対して発言があった場合に、ここで定義されている会話文との類似度を計算して、アクションの判定が行われます。
    • 複数会話文を設定することが可能です。
      • 例）「注文方法について」「どうやって注文するの」と設定した場合、どちらの言い回しであっても同一のアクションとして判定されます。
    • 表記揺れや単語の類似度も考慮されています。
      • 「どうやって注文するの」と「どのように注文しますか」などの言い回しの違いも考慮しているため、別の会話文として登録する必要はないです。
      • 「PC」と「パソコン」」など別の単語でも似た意味として自動で認識されるため、別の会話文として登録する必要はないです。

このアクションに対する応答文タブ

  • このアクションに対する応答文:
    • 指定されたアクションに対する応答文の設定を行います。
    • チャットボットの応答は、１回の応答で複数メッセージ応答することができます。これを応答文グループと呼びます。
    • 複数の応答文グループを追加すると、応答文グループの中からランダムで１つ選択されて応答します。
    • 応答文の中にエンティティの値やステートで取得した情報を埋め込むことができます。詳しくはを応答文への埋込み参照
  • 応答タイプ:
    • 【Webの場合】
      応答タイプ説明
      Web - テキストメッセージ テキストで応答します。
      文字だけでなく、画像タグやビデオタグを埋め込むことも可能です。
      応答選択肢を指定することで、ユーザの次の発言の選択肢をボタンとして返すこともできます。
    • 【LINEの場合】
      応答タイプ説明
      LINE - テキストメッセージ テキストで応答します。
      LINE - イメージメッセージ 画像で応答します。
      LINE - イメージマップメッセージ 画像で応答します。
      画像内にタップエリアを指定して、タップされた場合の行動（ウェブサイトを開くもしくはメッセージの送信）を指定することができます。
      LINE - 確認メッセージ LINEの確認タイプのメッセージで応答します。
      ２つのボタンを指定できます。
      LINE - ボタンメッセージ LINEのボタンタイプのメッセージで応答します。
      画像の指定や４つまでボタンの指定ができます。
      LINE - カルーセルメッセージ LINEのカルーセルタイプのメッセージで応答します。
      カラム毎にボタンメッセージを指定することができます。
      カラムは５つまで設定が可能です。
    • 【Facebookの場合】
      応答タイプ説明
      Facebook - テキストメッセージ テキストで応答します。
      Facebook - イメージメッセージ 画像で応答します。
      Facebook - ボタンメッセージ Facebookのボタンタイプのメッセージで応答します。
      3つまでボタンの指定ができます。
      Facebook - カルーセルメッセージ Facebookのカルーセルタイプのメッセージで応答します。
      カラム毎にボタンメッセージを指定することができます。
      カラムは10個まで設定が可能です。
