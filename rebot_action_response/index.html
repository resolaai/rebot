応答文への情報の埋込み

応答文の中にエンティティの値やステートで取得した情報を埋め込むことができます。

ここでは、フォーム穴埋めタイプで取得した情報を応答文へ埋め込む方法を説明します。
（ステートを理解していない場合は、ステートの項目を先に読むことをお薦めします）

サンプル

以下は、フォーム穴埋めタイプのステートで返品対応を行い、必要な情報を全て集めた状態です。
このようにRebotでは、現在の状態をJSON形式で保有しており、応答文からこれらの情報にアクセスすることが可能です。

{
  "action": "sys_no_action",
  "state": {
      "return": {
          "form_next": "",
          "form": {
              "email": "test@test.com",
              "return_reason": "思ったものと違った",
              "return_method": {
                  "origin": "返金",
                  "value": "返金"
              },
              "return_product": {
                  "origin": "商品xx-x",
                  "value": "商品XX-X"
              }
          },
          "status": [
              "confirm",
          ]
      }
  },
  "text": "test@test.com",
  "entity": {},
}

ここで、応答文として、以下のような返答をしたい場合を考えます。

商品「商品XX-X」の返品でよろしいですね。
----------------------------
返品理由：思ったものと違った
手続き：返金
EMAIL：test@test.com
----------------------------

この場合、「商品XX-X」「思ったものと違った」「返金」「test@test.com」がステートで取得した値です。
応答文でこれらの値を表示する為には、上記JSONの値にドット(.)区切りでアクセスするパスを指定します。

商品「{{state.return.form.return_product.value}}」の返品でよろしいですね。
----------------------------
返品理由：{{state.return.form.return_reason}}
手続き：{{state.return.form.return_method.value}}
EMAIL：{{state.return.form.email}}
----------------------------

これで応答文にステートで取得した値を埋め込むことができるようになります。
